# Eva OS Git Versioning & Workflow (GIT.MD)

Этот репозиторий использует строгую модель ветвления и семантического версионирования для поддержания чистоты кода и изоляции экспериментов.

## Версионирование (SemVer: `x.y.z`)

Формат версии: `x.y.z`
*   **x (Major)**: Мажорное обновление, меняющее архитектуру Ядра. Всегда находится в ветке `main`.
*   **y (Minor)**: Минорное обновление, добавление новых фишек и модулей (например, UI, бенчмарки). Сливается в `develop`.
    *   *Правило разработки*: В процессе работы над фишкой в изолированной ветке, коммиты версионируются буквами. Например: `1.1.0-a`, `1.1.0-b`. Если буквы заканчиваются — `1.1.0-aa`.
*   **z (Patch/Bugfix)**: Мелкие исправления багов (хотфиксы).
    *   *Правило*: Баги в процессе разработки фишки исправляются прямо в её ветке. Отдельные ветки с префиксом `fix-` создаются от `develop` *только* для исправления багов, найденных пользователями в продакшене.

## Ветки (Branches)

1.  **`main`**: Продакшен. Всегда стабилен. Сюда попадают только протестированные релизы из `develop`.
2.  **`develop`**: Главная ветка разработки. Сюда сливаются все завершенные фишки.
3.  **Фиче-ветки (`feature/*` или `x.y.0-*`)**: Отпочковываются от `develop`. Здесь ведется вся грязная работа (например, `1.1.0-ui`).
4.  **Багфиксы (`fix-*`)**: Отпочковываются от `develop`, сливаются обратно в `develop` (и затем патчем в `main`).

## CI/CD и Артефакты

*   **Линтер и Тесты (cargo test / cargo clippy / pre-commit)**: Отрабатывают на **всех** ветках при каждом коммите.
*   **Сборка бинарников (Artifacts)**:
    *   Собираются только в ветке `develop` (с тегом `x.y.z-testbuild`).
    *   Собираются в релиз ветки `main` (с тегом `x.y.z`).
    *   В фиче-ветках артефакты не собираются для экономии ресурсов CI.
